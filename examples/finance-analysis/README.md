# Example: Finance Analysis

This example shows how to configure DQIII8 as a financial analysis assistant.

## What this example covers

- WACC and DCF valuation workflows
- Ratio analysis from financial statements
- Backtesting trading strategies
- Generating executive reports in Markdown or PDF

## Project setup

```bash
# Set the active project
export JARVIS_PROJECT=finance-analysis

# Use the finance tier (Claude API for complex reasoning)
export JARVIS_MODEL=claude-sonnet-4-6

# Start a session
claude
```

## Example requests

```
"Analyze Apple's WACC and generate an executive report"
"Backtest a momentum strategy on BTC over 3 years"
"Calculate DCF valuation for MSFT with 10% discount rate"
"Compare Sharpe ratios across S&P 500 sectors"
```

## How DQIII8 routes finance tasks

Finance and trading requests automatically route to **Tier 3** (Claude API) via the model router:

```
task_type=finance  → agent=finance-analyst  → tier=3 → claude-sonnet-4-6
task_type=trading  → agent=quant-analyst    → tier=3 → claude-sonnet-4-6
```

## Output formats

- `report` — Markdown executive summary
- `pdf` — Rendered PDF (requires pandoc)
- `code` — Python analysis script

## Adding domain knowledge

Place reference documents in `.claude/agents/finance-analyst/knowledge/`:

```
.claude/agents/finance-analyst/knowledge/
├── valuation_methods.md
├── ratio_benchmarks.md
└── index.json          # auto-generated by knowledge_indexer.py
```

Re-index after adding files:

```bash
python3 bin/knowledge_indexer.py --agent finance-analyst
```
